The Cross-Validation Set








Typically, there are two kinds of sets in a dataset.
  • Training Set : The bigger chunk of our dataset on which we train our machine learning algorithm.
  • Test Set : The other piece of our dataset on which we test the performance of our machine learning algorithm.

Well imagine a scenario. You trained your algorithm on the training set and used the test set to check the performance of your algorithm. As it turns out, your algorithm's performance on the test set was very poor. Now you thought if I train my model for a bit longer, it might solve my problem. You increased the number of epochs for training and it slightly improved peroformance over test set, but this isn't satisfactory yet. Now it occured to you, how about using another machine learning algorithm, or using another activation function or using or adding regularization or tweaking some other hyperparameter.
In the above process, what happened was that you un-intentionally trained your algorithm over the test set and your test set is no more a real test set. It doesn't anymore projects the real scenario to you.

So what people do to avoid this condition is they make another set, specifically to tweak and fool around with the hyperparameters, and this set is called the Cross-Validation Set.
